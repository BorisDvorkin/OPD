Ядро - это набор подсистем, управдяющих оборудованием и программами пользователя.
Подсистемы - память, процессы, потоки, файловая система, сетевая подсистема, драйвера и платформо-зависимый код

исполняемая программа - адресное пространство, разитое на несколько сегментов

Файл "." - это ссылка директории на саму себя
Файл ".." - это ссылка на директорию выше по файловой иерархии(на одну. т.е. это родительская директория)

Абсолютный путь - путь от корня файловой системы
Относительный путь - путь от текущей директории, откуда была запущена программма пользователя

> file : перенаправление stdout в file
>> file - добавить stdout к file
2> file - пеернаправить stderr в file
2>> file - добавить stderr к file
< file - взять stdin из file
<< EOF - записать в stdin из терминала до символов "EOF"
| - перенапрааление stdout первой команды на stdin второй команды

Отличие xargs и | - в том, что не все файлы читают standart input
к примеру, команды echo и ls не умебт.
Поэтому надо для них использовать xargs.
например, если хотим передать файлы ls, то нужно написать перед этим xargs.

Триггер - это электронная схема, выполняющая функционал защёлки, с помощью которой можно хранить и управлять информацией
D-триггер - ячейка, хранящая один бит иинформации. У него есть два входа:
D - информационный
C - вход синхронизации

Триггеры объединяются в регистры
Дешифратор - это устройство, на ввод которого подается код числа, а на выходе выбирается только одна выходная линия, номер которой соответствует коду на входе дешифратора.
Адресуемая память - это память, которая выбирает одну из ячеек, которая соответствует коду адреса, после чего операция с памятью производится с этой выбранной ячейкой.
